## 1. Descripción del Modelo

El estándar **3GPP TR 38.901** define los modelos de canal para frecuencias de 0.5 a 100 GHz, desarrollados para 5G NR (New Radio). Este es el modelo de referencia de la industria para planificación de redes 5G.

A diferencia de los modelos empíricos clásicos (Okumura-Hata, P.1546), el TR 38.901 usa un enfoque **probabilístico**: cada punto receptor tiene una probabilidad P_LOS(d) de estar en condición LOS, y la pérdida total es la combinación ponderada de las pérdidas LOS y NLOS.

### 1.1 Escenarios Definidos

| Escenario | Nombre | Descripción |
|-----------|--------|-------------|
| `UMa` | Urban Macro | Celda urbana macro, antena sobre nivel de techos |
| `UMi` | Urban Micro | Celda urbana micro/small cell, antena bajo nivel de techos |
| `RMa` | Rural Macro | Celda rural macro, largo alcance |

### 1.2 Tipo de Modelo

| Aspecto | Descripción |
|---------|-------------|
| Tipo | Probabilístico estocástico |
| Condición LOS/NLOS | Probabilística — P_LOS(d) continua |
| Uso de terreno | Opcional (modo `use_dem=True`) — difracción efectiva ITU-R P.526 (knife-edge) |
| Vectorización | CPU (NumPy) / GPU (CuPy) via `self.xp` |
| Referencia | 3GPP TR 38.901 v17.0.0 (2022) |

### 1.3 Rangos de Validez por Escenario

| Escenario | Frecuencia (GHz) | Distancia (m) | h_bs típica |
|-----------|-----------------|--------------|-------------|
| UMa | 0.5–100 | 10–10000 | 25 m |
| UMi | 0.5–100 | 10–5000 | 10 m |
| RMa | 0.5–100 | 10–10000 | 35 m |

## 4. Probabilidad LOS — P_LOS(d)

### 4.1 Fórmula

$$P_{\text{LOS}}(d) = \min\!\left(\frac{C_1}{d}, 1\right) \cdot \left(1 - e^{-d/C_2}\right) + e^{-d/C_2}$$

donde:
- $d$ = distancia en **metros**
- $C_1, C_2$ = coeficientes específicos del escenario (ver tabla §3.1)

**Código real:**
```python
# three_gpp_38901.py, método _calculate_los_probability()
C1 = scenario_params['los_prob_coeff1']
C2 = scenario_params['los_prob_coeff2']

term1 = self.xp.minimum(C1 / distances_m, 1.0)
term2 = 1.0 - self.xp.exp(-distances_m / C2)
term3 = self.xp.exp(-distances_m / C2)

los_prob = term1 * term2 + term3
los_prob = self.xp.clip(los_prob, 0.0, 1.0)
```

### 4.2 Análisis de la Función

La función $P_{\text{LOS}}(d)$ tiene dos regímenes:

- **d muy pequeña** → `min(C1/d, 1) → 1`, `exp(−d/C2) → 1` → P_LOS → 1 (LOS casi seguro)
- **d muy grande** → `min(C1/d, 1) → 0`, `exp(−d/C2) → 0` → P_LOS → 0 (NLOS probable)

La transición ocurre alrededor de d ≈ C1 (metros):
- UMa: transición ≈ 18 m
- UMi: transición ≈ 21 m
- RMa: transición ≈ 21 m

### 4.3 Valores Numéricos de P_LOS (UMa, C1=18, C2=63)

| d (m) | min(18/d,1) | exp(−d/63) | P_LOS |
|-------|------------|-----------|-------|
| 10 | 1.000 | 0.855 | 1.000 |
| 18 | 1.000 | 0.752 | 1.000 |
| 50 | 0.360 | 0.453 | 0.616 |
| 100 | 0.180 | 0.205 | 0.332 |
| 200 | 0.090 | 0.042 | 0.129 |
| 500 | 0.036 | 0.000 | 0.036 |
| 1000 | 0.018 | 0.000 | 0.018 |

## 5. Path Loss LOS

$$\text{PL}_{\text{LOS}} = C_{0,\text{LOS}} + C_{1,\text{LOS}}\,\log_{10}(d_m) + 20\,\log_{10}(f_{\text{GHz}}) \quad \text{[dB]}$$

donde $d_m$ = distancia en **metros**.

**Código real:**
```python
# three_gpp_38901.py, método _calculate_path_loss_los()
C0 = scenario_params['los_path_loss_intercept']
C1 = scenario_params['los_path_loss_distance_slope']

pl_los = C0 + C1 * self.xp.log10(distances_m) + 20 * self.xp.log10(f_ghz)
```

**Ejemplo UMa, 3.5 GHz, d=200 m:**
```
PL_LOS = 28.0 + 22·log10(200) + 20·log10(3.5)
       = 28.0 + 22·2.301 + 20·0.544
       = 28.0 + 50.62 + 10.88
       = 89.5 dB
```

**Justificación del exponente C1=22:** En LOS con dispersión cercana a 2 (Free Space tiene C1=20), el coeficiente 22 captura la atenuación adicional por scattering atmosférico y difracción leve sobre obstáculos.

---

## 6. Path Loss NLOS

$$\text{PL}_{\text{NLOS}} = C_{0,\text{NLOS}} + C_{1,\text{NLOS}}\,\log_{10}(d_m) + 20\,\log_{10}(f_{\text{GHz}}) + C_2\,(h_{\text{ue}} - 1.5) \quad \text{[dB]}$$

donde:
- $h_{\text{ue}}$ = altura del receptor en metros
- $C_2 = -0.6$ dB/m (uniforme en todos los escenarios)
- Referencia de h_ue: 1.5 m

**Código real:**
```python
# three_gpp_38901.py, método _calculate_path_loss_nlos()
C0 = scenario_params['nlos_path_loss_intercept']
C1 = scenario_params['nlos_path_loss_distance_slope']
C2 = scenario_params['height_correction']  # = -0.6 dB/m

pl_nlos = (C0
           + C1 * self.xp.log10(distances_m)
           + 20 * self.xp.log10(f_ghz)
           + C2 * (h_ue - 1.5))
```

**Ejemplo UMa, 3.5 GHz, d=200 m, h_ue=1.5 m:**
```
PL_NLOS = 13.54 + 39.08·log10(200) + 20·log10(3.5) + (−0.6)·(1.5−1.5)
        = 13.54 + 39.08·2.301 + 10.88 + 0
        = 13.54 + 89.94 + 10.88
        = 114.4 dB
```

**Diferencia LOS/NLOS a 200 m, UMa:** 114.4 − 89.5 = **24.9 dB**

### 6.1 Corrección por Altura UE

$$\Delta_{\text{height}} = C_2 \cdot (h_{\text{ue}} - 1.5) = -0.6 \cdot (h_{\text{ue}} - 1.5) \quad \text{[dB]}$$

| h_ue | Δ_height | Interpretación |
|------|---------|---------------|
| 0 m (suelo) | +0.9 dB | Más pérdida (por debajo de referencia) |
| 1.5 m (ref) | 0 dB | Referencia estándar |
| 3 m (escritorio) | −0.9 dB | Menor pérdida |
| 10 m (edificio) | −5.1 dB | Ganancia significativa por altura |

## 7. Path Loss Final — Mezcla Probabilística

$$\text{PL}_{\text{total}}(d) = P_{\text{LOS}}(d) \cdot \text{PL}_{\text{LOS}}(d) + \bigl[1 - P_{\text{LOS}}(d)\bigr] \cdot \text{PL}_{\text{NLOS}}(d)$$

**Código real:**
```python
# three_gpp_38901.py, líneas 233-234
los_prob = self._calculate_los_probability(distances_m)
pl_los   = self._calculate_path_loss_los(f_ghz, distances_m, h_ue)
pl_nlos  = self._calculate_path_loss_nlos(f_ghz, distances_m, h_ue)

path_loss = los_prob * pl_los + (1.0 - los_prob) * pl_nlos
```

**Justificación:** Este enfoque es matemáticamente equivalente a calcular la pérdida esperada sobre el conjunto de realizaciones estocásticas del canal. No asigna un estado binario LOS/NLOS a cada punto, sino que interpola suavemente entre ambas condiciones según la probabilidad estadística calibrada con mediciones reales.

**Ejemplo a varios puntos, UMa, 3.5 GHz:**

| d (m) | P_LOS | PL_LOS (dB) | PL_NLOS (dB) | PL_total (dB) |
|-------|-------|------------|-------------|--------------|
| 50 | 0.616 | 81.5 | 102.4 | 89.8 |
| 100 | 0.332 | 89.5 | 114.4 | 106.2 |
| 200 | 0.129 | 96.4 | 121.0 | 117.9 |
| 500 | 0.036 | 106.5 | 132.2 | 130.9 |
| 1000 | 0.018 | 113.2 | 141.0 | 139.3 |

## 8. Corrección por Terreno (modo use_dem=True)

Cuando se activa `use_dem=True`, el modelo añade una corrección de difracción basada en **ITU-R P.526 (single knife-edge)** usando un perfil efectivo sobre el grid DEM.

### 8.1 Ecuaciones Usadas (P.526)

Se define el parámetro de difracción:

$$v = h \sqrt{\frac{2(d_1 + d_2)}{\lambda d_1 d_2}}$$

donde:
- $h$ = intrusión del obstáculo sobre la LOS (m)
- $d_1, d_2$ = distancias TX→obstáculo y obstáculo→RX (m)
- $\lambda = c/f$ = longitud de onda (m)

La pérdida adicional por difracción se calcula como:

$$L_d(v) =
\begin{cases}
0, & v \le -0.78 \\
6.9 + 20\log_{10}\left(\sqrt{(v-0.1)^2 + 1} + v - 0.1\right), & v > -0.78
\end{cases}$$

En la implementación, el término final se acota con:

$$L_d \leftarrow \min(L_d, L_{\max})$$

con $L_{\max} = 40$ dB por defecto (`max_terrain_correction_db`).

### 8.2 Perfil Efectivo sobre DEM

No se hace ray-tracing completo; se usa un perfil efectivo vectorizado:

1. Se identifica el píxel TX como el de menor distancia en `distances_m`.
2. Para cada receptor, se muestrean `dem_profile_samples` puntos sobre la recta TX→RX.
3. Se calcula la LOS entre:
   - altura TX absoluta: `tx_elevation + h_bs`
   - altura RX absoluta: `terrain_heights + h_ue`
4. Se obtiene el obstáculo dominante como el máximo despeje positivo sobre la LOS.
5. Se aplica P.526 para obtener $L_d$ por píxel.

### 8.3 Acoplamiento con Mezcla Probabilística 3GPP

Para no romper la naturaleza probabilística del modelo 38.901, la corrección se pondera con la probabilidad NLOS:

$$\Delta PL_{terrain} = L_d \cdot (1 - P_{LOS})$$

y el path loss final queda:

$$PL_{final} = PL_{3GPP\_blend} + \Delta PL_{terrain}$$

### 8.4 Código Real Implementado

```python
# three_gpp_38901.py, método _apply_terrain_correction()
v = h_obs * xp.sqrt((2.0 * (d1 + d2)) / (wavelength_m * d1 * d2))

knife_loss = xp.where(
     v <= -0.78,
     0.0,
     6.9 + 20.0 * xp.log10(xp.sqrt((v - 0.1) ** 2 + 1.0) + v - 0.1)
)

knife_loss = xp.maximum(knife_loss, 0.0)
knife_loss = xp.minimum(knife_loss, self.max_terrain_correction_db)
correction = knife_loss * xp.clip(1.0 - los_prob, 0.0, 1.0)
```

### 8.5 Advertencia Importante

```
UserWarning: "3GPP terrain correction mode uses an ITU-R P.526 knife-edge 
effective profile approximation."
```

La formulación de difracción sí es estándar P.526; la aproximación está en el perfil efectivo (muestreo sobre grid), no en la ecuación de pérdida.

### 8.6 Distancia de Breakpoint

El estándar define una distancia de quiebre donde el comportamiento del path loss cambia:

$$d_{\text{BP}} = \frac{4\,h_{\text{bs}}\,h_{\text{ue}}\,f_{\text{GHz}}}{c} \approx 4\,h_{\text{bs}}\,h_{\text{ue}} \times \frac{f_{\text{GHz}}}{10^9/10^9} \cdot \frac{1}{0.3}$$

**Implementación simplificada:**
```python
# three_gpp_38901.py, método get_breakpoint_distance()
d_bp = 4 * self.h_bs * self.h_ue * 0.3   # en metros (c ≈ 0.3 m/ns·GHz)
```

```

         Config: scenario, h_bs, h_ue, use_dem

     │
     ▼
┌────────────────────────────────────────────────────────┐
│ 1. Resolver alturas (cascada de prioridades)          │
│    h_bs: kwargs > self.h_bs > tx_height              │
│    h_ue: kwargs > self.h_ue > rx_height              │
└────────────────────────────────────────────────────────┘
     │
     ▼
┌────────────────────────────────────────────────────────┐
│ 2. Conversión MHz → GHz                              │
│    f_ghz = frequency / 1000                          │
└────────────────────────────────────────────────────────┘
     │
     ▼
┌────────────────────────────────────────────────────────┐
│ 3. Convertir distances a array                        │
│    distances_m = asarray(distances)   ← en METROS   │
└────────────────────────────────────────────────────────┘
     │
     ▼
┌────────────────────────────────────────────────────────┐
│ 4. P_LOS(d) — probabilidad LOS por distancia         │
│    C1, C2 = SCENARIOS[scenario][...]                 │
│    P_LOS = min(C1/d,1)·(1−e^−d/C2) + e^−d/C2       │
└────────────────────────────────────────────────────────┘
     │
     ▼
┌────────────────────────────────────────────────────────┐
│ 5. PL_LOS = C0_L + C1_L·log10(d_m) + 20·log10(f_G) │
└────────────────────────────────────────────────────────┘
     │
     ▼
┌────────────────────────────────────────────────────────┐
```

## 11. Ejemplo Numérico Completo — Escenario UMi, 5G, 3.5 GHz

**Escenario:** Small cell en Cuenca centro, 3.5 GHz

```
Configuración:
  scenario   = 'UMi'
  h_bs       = 8 m (poste en calle)
  h_ue       = 1.5 m (smartphone)
  frequency  = 3500 MHz → f_ghz = 3.5
  use_dem    = False

Coeficientes UMi:
  C1_P=21, C2_P=109.5
  C0_LOS=32.4, C1_LOS=21
  C0_NLOS=35.3, C1_NLOS=40
  C2=−0.6

Para d = 150 m:

P_LOS:
  min(21/150, 1) = 0.140
  exp(−150/109.5) = exp(−1.37) = 0.254
  P_LOS = 0.140·(1−0.254) + 0.254
        = 0.140·0.746 + 0.254
        = 0.104 + 0.254 = 0.358

PL_LOS:
  = 32.4 + 21·log10(150) + 20·log10(3.5)
  = 32.4 + 21·2.176 + 20·0.544
  = 32.4 + 45.70 + 10.88
  = 88.98 dB

PL_NLOS (h_ue=1.5):
  = 35.3 + 40·log10(150) + 20·log10(3.5) + (−0.6)·(1.5−1.5)
  = 35.3 + 40·2.176 + 10.88 + 0
  = 35.3 + 87.04 + 10.88
  = 133.22 dB

PL_total:
  = 0.358·88.98 + (1−0.358)·133.22
  = 31.85 + 85.53
  = 117.4 dB
```

## 14. Cómo lo Invoca CoverageCalculator

```python
# src/core/coverage_calculator.py

# El modelo recibe distancias en METROS (importante: no km)
path_loss_args = {
    'distances':       distances,        # ← METROS, shape (100, 100)
    'frequency':       antenna.frequency_mhz,  # MHz (se convierte a GHz internamente)
    'tx_height':       antenna.height_agl,
    'terrain_heights': terrain_heights,  # opcional (solo si use_dem=True)
     'tx_elevation':    tx_elevation,     # cota terreno TX en msnm
    # Parámetros del escenario vía model_params:
    'h_bs': 25.0,    # altura BS configurada en la simulación
    'h_ue': 1.5,     # altura UE
}
path_loss = gpp_model.calculate_path_loss(**path_loss_args)
```

> **Diferencia clave:** Mientras que Okumura-Hata, COST-231 y P.1546 dividen `distances/1000` internamente, el modelo 3GPP recibe las distancias en **metros** y trabaja en metros. La confusión con unidades fue un bug histórico en el proyecto (documentado en `docs/`).
