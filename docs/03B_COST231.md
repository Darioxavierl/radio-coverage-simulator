## 1. Descripción del Modelo

El modelo **COST-231 Walfisch-Ikegami** es un modelo **semi-determinístico** para propagación en entornos urbanos. Combina:

- **Modelo de Ikegami (1984):** Difracción sobre la última fila de edificios hacia la calle (Lrtd)
- **Modelo de Walfisch (1988):** Difracción multi-pantalla sobre filas de edificios (Lmsd)
- **Proyecto COST-231 (1999):** Unificación y calibración con factor de entorno

A diferencia de Okumura-Hata (puramente empírico), este modelo incorpora parámetros físicos del entorno urbano: altura de edificios, ancho de calles y orientación de las calles.

### 1.1 Tipo de Modelo

| Aspecto | Descripción |
|---------|-------------|
| Tipo | Semi-determinístico (empírico + geometría urbana) |
| Condición LOS/NLOS | Sí — heurística de altura efectiva |
| Uso de terreno | Sí — altura efectiva TX sobre DEM |
| Vectorización | CPU (NumPy) / GPU (CuPy) via `self.xp` |
| Referencia | COST-231 Final Report, cap. 4; ITU-R P.1411-8 |

### 1.2 Rangos de Validez

| Parámetro | Mínimo | Máximo | Unidades |
|-----------|--------|--------|----------|
| Frecuencia | 800 | 2000 | MHz |
| Distancia | 20 | 5000 | m (0.02–5 km) |
| Altura TX (h_bs) | 30 | 200 | m AGL |
| Altura móvil (h_m) | 1 | 10 | m AGL |

## 3. Integración con el Terreno (DEM)

### 3.1 Altura Efectiva del TX

Al igual que Okumura-Hata, el modelo calcula la altura efectiva de la antena base sobre el nivel promedio del terreno:

$$h_{bs,\text{eff}} = h_{\text{ant}} + h_{\text{elev,TX}} - \overline{h_{\text{terreno}}}$$

**Código real:**
```python
# cost231.py, líneas 112-116
terrain_avg = self.xp.mean(terrain_heights_flat)
h_bs_eff = tx_height + tx_elevation - terrain_avg
h_bs_eff = self.xp.maximum(h_bs_eff, 30.0)   # mínimo 30 m
h_bs_eff = self.xp.minimum(h_bs_eff, 200.0)  # máximo 200 m
```

### 3.2 Diferencias de Altura Derivadas

A partir de `h_bs_eff` se calculan dos diferencias clave para la geometría del urban canyon:

$$\Delta h_{rm} = h_{bs,\text{eff}} - h_{\text{building}}$$
$$\Delta h_{ms} = h_{\text{building}} - h_{\text{mobile}}$$

- $\Delta h_{rm}$ = cuánto sobresale la antena base sobre la línea de techos (rooftop)
- $\Delta h_{ms}$ = cuánto está la línea de techos por encima del receptor

**Código real:**
```python
# cost231.py, líneas 130-132
delta_h_rm_array = self.xp.full_like(distances_km, h_bs_eff - building_height)
delta_h_ms_array = self.xp.full_like(distances_km, building_height - mobile_height)
```

**Ejemplo:** h_bs_eff=75 m, building_height=15 m, mobile_height=1.5 m
- Δh_rm = 75 − 15 = 60 m (antena muy por encima de los techos)
- Δh_ms = 15 − 1.5 = 13.5 m (receptores en la calle)

## 4. Determinación LOS/NLOS

### 4.1 Heurística Implementada

El modelo determina LOS/NLOS con base en la diferencia entre la cota de la antena y la cota promedio del terreno:

$$\delta_h = h_{\text{ant}} + h_{\text{elev,TX}} - \overline{h_{\text{terreno}}}$$

$$\text{LOS} \iff \delta_h > 30 \text{ m}$$

**Código real:**
```python
# cost231.py, método _determine_los_nlos()
h_tx_eff = tx_height + tx_elevation
terrain_avg = self.xp.mean(terrain_heights)
delta_h = h_tx_eff - terrain_avg

# 30 m = umbral típico para urban canyon (ITU-R P.1411)
los_mask = self.xp.full(distances.shape, delta_h > 30.0, dtype=bool)
```

El resultado es un **array de booleanos** del mismo shape que `distances`. La decisión LOS/NLOS es **uniforme** para toda la grilla — si la antena sobresale >30 m sobre el terreno promedio, todos los puntos se tratan como LOS.

### 4.2 Justificación del Umbral 30 m

En entornos urbanos densos (referencia ITU-R P.1411), una antena que sobresale al menos 30 m sobre el nivel promedio de los techos tiene probabilidad alta de tener línea de vista hacia la mayoría de los receptores. Por debajo de este umbral, la difracción multi-pantalla domina.

### 4.3 Efecto sobre el Cálculo

- **LOS:** PL = PL_base + Lrtd + Cf (sin Lmsd)
- **NLOS:** PL = PL_base + Lrtd + Lmsd + Cf

```python
# cost231.py, líneas 145-150
lmsd = self.xp.zeros_like(distances_km)
if self.xp.any(~los_mask):
    lmsd[~los_mask] = self._calculate_lmsd(
        frequency, distances_km[~los_mask],
        delta_h_ms_array[~los_mask], environment
    )
# Solo los puntos NLOS reciben contribución Lmsd
```

## 5. Ecuaciones Implementadas

### 5.1 Path Loss Base (Free Space)

$$L_0 = 32.45 + 20\,\log_{10}(f_{\text{MHz}}) + 20\,\log_{10}(d_{\text{km}})$$

Este es el mismo baseline de espacio libre, válido para LOS ideal. Se aplica tanto en LOS como en NLOS; los términos de difracción se suman encima.

**Código real:**
```python
# cost231.py, método _calculate_base_path_loss()
pl = (32.45
      + 20.0 * self.xp.log10(frequency)
      + 20.0 * self.xp.log10(distances_km))
```

### 5.2 Pérdida por Difracción Rooftop-to-Street (Lrtd)

Representa la difracción de la señal desde el último techo hasta el receptor en la calle. Depende del ancho de calle, la altura relativa del TX respecto a los techos, y la orientación de la calle.

$$L_{\text{rtd}} = -16.9 - 10\,\log_{10}(w) + 10\,\log_{10}(f) + 20\,\log_{10}(\Delta h_{rm}) + L_{\text{ori}}$$

donde:
- $w$ = ancho de calle en **metros**
- $f$ = frecuencia en **MHz**
- $\Delta h_{rm}$ = `max(h_bs_eff − h_building, 0.1)` en **metros**
- $L_{\text{ori}}$ = factor de orientación de calle (ver §5.3)

**Código real:**
```python
# cost231.py, método _calculate_lrtd()
delta_h_rm = self.xp.maximum(delta_h_rm, 0.1)   # protección log(0)
street_width = self.xp.maximum(street_width, 0.5)

lrtd = (-16.9
        - 10.0 * self.xp.log10(street_width)
        + 10.0 * self.xp.log10(frequency)
        + 20.0 * self.xp.log10(delta_h_rm)
        + lori)
```

**Justificación de constantes:**
- **−16.9:** Constante de calibración Ikegami para geometría típica del urban canyon
- **−10·log₁₀(w):** Calle más ancha → más espacio libre → menor difracción
- **+10·log₁₀(f):** Mayor frecuencia → mayor difracción en bordes de techo
- **+20·log₁₀(Δh_rm):** Mayor sobresalida TX sobre techos → mayor ángulo de incidencia → mayor difracción en esquina

### 5.3 Factor de Orientación de Calle — Lori

La orientación relativa entre la calle y la dirección TX→RX afecta significativamente la difracción en el borde del techo.

$$L_{\text{ori}} = \begin{cases}
-10 + 0.354\,\varphi & 0° < \varphi \leq 35° \\
2.5 + 0.075\,(\varphi - 35) & 35° < \varphi \leq 55° \\
4.0 - 0.114\,(\varphi - 55) & 55° < \varphi \leq 90°
\end{cases}$$

donde $\varphi = |\text{street\_orientation}| \mod 90°$

**Código real:**
```python
# cost231.py, método _calculate_orientation_factor()
phi = abs(street_orientation) % 90.0

if phi <= 35.0:
    lori = -10.0 + 0.354 * phi
elif phi <= 55.0:
    lori = 2.5 + 0.075 * (phi - 35.0)
else:
    lori = 4.0 - 0.114 * (phi - 55.0)
```

**Rango de Lori:**
- φ=0° (calle perpendicular a TX): Lori = −10 dB (mínima difracción)
- φ=55° (calle diagonal 55°): Lori = +4.0 dB (máxima difracción)
- φ=90° (calle paralela a TX): Lori = −0.04 dB ≈ 0 dB

### 5.4 Pérdida por Difracción Multi-Pantalla — Lmsd (solo NLOS)

Representa la difracción acumulada sobre múltiples filas de edificios entre TX y RX. Solo se aplica en condición NLOS.

$$L_{\text{msd}} = -18\,\log_{10}(1 + \Delta h_{ms}) + 10\,\log_{10}(f) + 10\,\log_{10}(d_{\text{km}}) + C_{\text{env}}$$

donde:
- $\Delta h_{ms}$ = `max(h_building − h_mobile, 0.1)` en **metros**
- $C_{\text{env}}$ = corrección por entorno (ver tabla)

| Entorno | C_env |
|---------|-------|
| Urban | −4 dB |
| Suburban | −8 dB |
| Rural | −12 dB |

**Código real:**
```python
# cost231.py, método _calculate_lmsd()
c_env_dict = {'Urban': -4.0, 'Suburban': -8.0, 'Rural': -12.0}
c_env = c_env_dict.get(environment, -4.0)

delta_h_ms = self.xp.maximum(delta_h_ms, 0.1)
distances_km = self.xp.maximum(distances_km, 0.001)

lmsd = (-18.0 * self.xp.log10(1.0 + delta_h_ms)
        + 10.0 * self.xp.log10(frequency)
        + 10.0 * self.xp.log10(distances_km)
        + c_env)
```

**Justificación:**
- **−18·log₁₀(1+Δh_ms):** Mayor diferencia techo-receptor → más obstrucción por filas de edificios → mayor pérdida
- **+10·log₁₀(f):** Mayor frecuencia → más atenuación por difracción múltiple
- **+10·log₁₀(d):** A mayor distancia hay más filas de edificios atravesadas

### 5.5 Corrección por Entorno — Cf

Factor de ajuste final que refleja las diferencias estadísticas entre entornos:

$$C_f = \begin{cases}
0 \text{ dB} & \text{Urban} \\
-2 - 5.4\,\log_{10}(f) & \text{Suburban} \\
-4.78\,(\log_{10} f)^2 + 18.33\,\log_{10}(f) - 40.94 & \text{Rural}
\end{cases}$$

**Código real:**
```python
# cost231.py, método _calculate_environment_correction()
log_f = np.log10(frequency)

if environment == 'Urban':
    cf = 0.0
elif environment == 'Suburban':
    cf = -2.0 - 5.4 * log_f
elif environment == 'Rural':
    cf = -4.78 * (log_f**2) + 18.33 * log_f - 40.94
```

**Ejemplo numérico a 900 MHz (log10(900)=2.954):**
| Entorno | Cf calculado |
|---------|-------------|
| Urban | 0 dB |
| Suburban | −2 − 5.4·2.954 = −17.95 dB |
| Rural | −4.78·(2.954)² + 18.33·2.954 − 40.94 = −28.5 dB |

### 5.6 Fórmula Final Completa

$$\text{PL}_{\text{LOS}} = L_0 + L_{\text{rtd}} + C_f$$

$$\text{PL}_{\text{NLOS}} = L_0 + L_{\text{rtd}} + L_{\text{msd}} + C_f$$

**Código real (ensamblaje):**
```python
# cost231.py, líneas 155-161
# LOS:  pl_loss = pl_base + lrtd + 0    + cf
# NLOS: pl_loss = pl_base + lrtd + lmsd + cf
pl_loss = pl_base + lrtd + lmsd + cf
# (lmsd = 0 para puntos LOS, ya inicializado a zeros)
```

## 6. Flujo de Cálculo Completo

```
Entrada: distances (m), frequency (MHz), tx_height (m),
         terrain_heights (msnm), building_height, street_width,
         street_orientation, mobile_height, environment

     │
     ▼
┌────────────────────────────────────────────────────┐
│ 1. Flatten + validación de parámetros             │
│    original_shape = distances.shape               │
│    distances_flat = ravel(distances)              │
└────────────────────────────────────────────────────┘
     │
     ▼
┌────────────────────────────────────────────────────┐
│ 2. Conversión y protección                        │
│    d_km = distances / 1000                        │
│    d_km = max(d_km, 0.00001)                     │
└────────────────────────────────────────────────────┘
     │
     ▼
┌────────────────────────────────────────────────────┐
│ 3. Altura efectiva TX                             │
│    terrain_avg = mean(terrain_heights)            │
│    h_bs_eff = tx_height + tx_elev − terrain_avg  │
│    h_bs_eff = clip(h_bs_eff, 30, 200)            │
└────────────────────────────────────────────────────┘
     │
     ▼
┌────────────────────────────────────────────────────┐
│ 4. LOS/NLOS                                       │
│    delta_h = (tx_height + tx_elev) − terrain_avg │
│    los_mask = (delta_h > 30)                     │
└────────────────────────────────────────────────────┘
     │
     ▼
┌────────────────────────────────────────────────────┐
│ 5. Δh_rm y Δh_ms                                 │
│    Δh_rm = h_bs_eff − building_height            │
│    Δh_ms = building_height − mobile_height        │
└────────────────────────────────────────────────────┘
     │
     ▼
┌────────────────────────────────────────────────────┐
│ 6. L_0 (Free Space)                              │
│    L_0 = 32.45 + 20·log(f) + 20·log(d_km)       │
└────────────────────────────────────────────────────┘
     │
     ▼
┌────────────────────────────────────────────────────┐
│ 7. Lori → Lrtd                                   │
│    lori = f(street_orientation)                  │
│    lrtd = −16.9 − 10·log(w) + 10·log(f)         │
│           + 20·log(Δh_rm) + lori                 │
└────────────────────────────────────────────────────┘
     │
     ▼
┌────────────────────────────────────────────────────┐
│ 8. Lmsd (solo puntos NLOS)                       │
│    lmsd[NLOS] = −18·log(1+Δh_ms) + 10·log(f)   │
│                 + 10·log(d) + C_env              │
│    lmsd[LOS] = 0                                 │
└────────────────────────────────────────────────────┘
     │
     ▼
┌────────────────────────────────────────────────────┐
│ 9. Cf (entorno)                                  │
│    Cf = 0 / función de log(f)                    │
└────────────────────────────────────────────────────┘
     │
     ▼
┌────────────────────────────────────────────────────┐
│ 10. PL = L_0 + Lrtd + Lmsd + Cf                 │
│     reshape a original_shape                      │
└────────────────────────────────────────────────────┘
     │
     ▼
Salida: path_loss (dB), shape = shape(distances)
```

## 7. Ejemplo Numérico Completo

**Escenario:** Antena urbana en Cuenca, condición NLOS, 1800 MHz

```
Parámetros:
  frequency        = 1800 MHz
  tx_height        = 20 m AGL
  tx_elevation     = 2560 msnm
  terrain_avg      = 2545 msnm
  building_height  = 12 m
  street_width     = 10 m
  street_orientation = 45°
  mobile_height    = 1.5 m
  distance         = 500 m (0.5 km)
  environment      = 'Urban'

Paso 3 — h_bs_eff:
  h_bs_eff = 20 + 2560 − 2545 = 35 m ✓ (30–200)

Paso 4 — LOS/NLOS:
  delta_h = (20 + 2560) − 2545 = 35 m > 30 m → LOS
  (Nota: en este ejemplo resulta LOS; si h_ant fuera menor, sería NLOS)

Supongamos que es NLOS para ejemplificar Lmsd (delta_h=25, NLOS):

Paso 5 — Δh:
  Δh_rm = 35 − 12 = 23 m
  Δh_ms = 12 − 1.5 = 10.5 m

Paso 6 — L_0 a 500 m = 0.5 km, 1800 MHz:
  L_0 = 32.45 + 20·log10(1800) + 20·log10(0.5)
      = 32.45 + 20·3.255 + 20·(−0.301)
      = 32.45 + 65.10 − 6.02
      = 91.53 dB

Paso 7 — Lori para φ=45°:
  35 < 45 ≤ 55 → Lori = 2.5 + 0.075·(45−35) = 2.5 + 0.75 = 3.25 dB

Paso 7 — Lrtd:
  Lrtd = −16.9 − 10·log10(10) + 10·log10(1800) + 20·log10(23) + 3.25
       = −16.9 − 10 + 32.55 + 27.24 + 3.25
       = 36.14 dB

Paso 8 — Lmsd (NLOS, Urban, C_env=−4):
  Lmsd = −18·log10(1+10.5) + 10·log10(1800) + 10·log10(0.5) + (−4)
       = −18·1.041 + 32.55 + (−3.01) + (−4)
       = −18.74 + 32.55 − 3.01 − 4
       = 6.80 dB

Paso 9 — Cf (Urban):
  Cf = 0 dB

Resultado NLOS:
  PL = 91.53 + 36.14 + 6.80 + 0 = 134.5 dB
```

## 8. Comparación LOS vs NLOS para el Mismo Escenario

| Condición | L_0 | Lrtd | Lmsd | Cf | **PL Total** |
|-----------|-----|------|------|----|--------------|
| LOS | 91.5 dB | 36.1 dB | 0 dB | 0 dB | **127.6 dB** |
| NLOS | 91.5 dB | 36.1 dB | 6.8 dB | 0 dB | **134.4 dB** |

Diferencia: ~6.8 dB — en práctica esto equivale a ~50% de reducción en potencia recibida.
