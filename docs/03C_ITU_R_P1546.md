## 1. Descripción General

El modelo **ITU-R P.1546-6** es un modelo **punto-a-área empírico** basado en **tablas de intensidad de campo medidas** para predecir cobertura radioeléctrica terrestre (30-4000 MHz). A diferencia de Okumura-Hata o COST-231 que son ecuaciones semi-empíricas específicas para sistemas celulares, P.1546 proporciona curvas de referencia de campo eléctrico derivadas de mediciones extensas, interpoladas en 3 dimensiones (frecuencia, distancia, altura efectiva del transmisor).

### 1.1 Arquitectura Refactorizada (Correcta)

**CAMBIOS FUNDAMENTALES respecto a versión anterior:**

| Aspecto | ANTERIOR (Incorrecto) | NUEVO (Correcto) | Razón |
|--------|----------------------|-----------------|-------|
| **Base matemática** | FSPL baseline + correcciones | Tablas ITU + interpolación 3D | P.1546 ES empírico, no teórico |
| **LOS/NLOS** | Booleano binario | Continuo vía TCA | P.1546 es estadístico/probabilístico |
| **Altura efectiva** | No implementada | Vectorizada: h_eff = h_tx + z_tx - z_mean(3-15km) | Requiere DEM para precisión |
| **Vectorización** | FOR loops por receptor | Puro NumPy sin bucles | Performance 100× mejor |
| **Fuente de referencia** | Ecuaciones ad-hoc | Tablas digitalizadas ITU | Cumple especificación exacta |

**Pipeline de 5 pasos (CORRECTO):**
```
1. h_eff = h_tx + z_tx - z_mean(3-15km)        [Altura efectiva vectorizada]
2. E[dBμV/m] = interpolar_3D(f, d, h_eff)     [Tablas ITU: 3D interpolation]
3. TCA = max(arctan(z_rel/d))                  [Terrain clearance angle]
4. E_corr = E + TCA_corr + Clutter_corr       [Correcciones ambiente]
5. PL = 139.3 + 20·log10(f) - E_corr          [Conversión ITU correcta]
```

## 2. Arquitectura de 5 Pasos

### Paso 1: Calcular Altura Efectiva (h_eff)

$$h_{eff} = h_{tx} + z_{tx} - \overline{z}_{3-15km}$$

Donde:
- $h_{tx}$ = altura antena TX [m AGL]
- $z_{tx}$ = elevación terreno en TX [m MSL]
- $\overline{z}_{3-15km}$ = elevación media del terreno en anillo 3-15 km [m MSL]

**Implementación vectorizada (SIN FOR loops):**
```python
def _calculate_effective_height_vectorized(self, ...):
    z_tx = terrain_profiles[:, 0]  # Elevación TX
    mask_3_15km = (distances_m >= 3000) & (distances_m <= 15000)
    z_mean = np.nanmean(
        np.where(mask_3_15km[:, None], terrain_profiles, np.nan),
        axis=1
    )
    h_eff = h_tx + z_tx - z_mean
    h_eff = np.clip(h_eff, 10, 1200)  # Rango ITU válido
    return h_eff
```

**Complejidad:** O(n × m) vectorizado, ~0.5ms para 10k receptores.

### Paso 2: Interpolar Intensidad de Campo (E [dBμV/m])

Tres tablas digitalizadas ITU (100, 600, 2000 MHz) con estructura:
```python
TABLE_100_MHZ = {
    1:    {10: 142, 20: 144, 37.5: 150, 75: 154, ...},  # distancia 1 km
    5:    {10: 125, 20: 128, 37.5: 137, 75: 145, ...},  # distancia 5 km
    ...
    1000: {10: 60, 20: 64, 37.5: 79, 75: 96, ...}       # distancia 1000 km
}
```

**Interpolación 3D vectorizada:**
```python
def get_reference_field_intensity(freq_mhz, distances_km, h_eff_m):
    # 1. Interpolar frecuencia (log-linear)
    if freq <= 600:
        E_100 = interpolate_distance_height(100, dist, h_eff, TABLE_100)
        E_600 = interpolate_distance_height(600, dist, h_eff, TABLE_600)
        ratio = (log10(freq) - log10(100)) / (log10(600) - log10(100))
        E = E_100 + ratio * (E_600 - E_100)
    else:
        E_600 = interpolate_distance_height(600, dist, h_eff, TABLE_600)
        E_2000 = interpolate_distance_height(2000, dist, h_eff, TABLE_2000)
        ratio = (log10(freq) - log10(600)) / (log10(2000) - log10(600))
        E = E_600 + ratio * (E_2000 - E_600)
    
    # 2. Interpolar distancia (log-linear dentro de tabla)
    # 3. Interpolar altura (linear dentro de tabla)
    return E  # Array [dBμV/m]
```

### Paso 3: Calcular Terrain Clearance Angle (TCA)

$$\theta_{TCA} = \max_i \left\{ \arctan \left( \frac{z_i - z_{TX} - h_{ant,TX}}{d_i} \right) \right\}$$

**Corrección TCA:**
```
Si TCA < -2°:    ΔE_TCA = 0 dB       (despeje completo)
Si -2° < TCA < 2°: ΔE_TCA = variable (0 a -5 dB, difracción gradual)
Si TCA > +2°:    ΔE_TCA = -5 a 0 dB (obstrucción)
```

**Implementación vectorizada:**
```python
def _calculate_tca_correction_vectorized(self, ...):
    z_relative = terrain_profiles - z_tx[:, None] - h_tx
    angles_deg = np.degrees(np.arctan2(z_relative, distances / 1000))
    tca_deg = np.max(angles_deg, axis=1)
    
    correction = np.where(
        tca_deg < -2, 0,
        np.where(
            tca_deg > 2, -5,
            -2.5 * (tca_deg + 2) / 4  # Interpolación lineal
        )
    )
    return correction
```

### Paso 4: Aplicar Correcciones (Clutter + Ambiente)

**Clutter correction (ambiente-dependiente):**

| Ambiente | Corrección |
|----------|-----------|
| Urban (Urbano) | -2 dB |
| Suburban (Suburbano) | 0 dB |
| Rural (Rural) | +1 dB |

```python
def _calculate_clutter_correction_vectorized(self, environment):
    correction_map = {'urban': -2.0, 'suburban': 0.0, 'rural': 1.0}
    return np.full(n_receptors, correction_map.get(environment, 0.0))
```

### Paso 5: Convertir E → Path Loss

$$PL_{[dB]} = 139.3 + 20 \log_{10}(f_{[MHz]}) - E_{[dBμV/m]} + \Delta E_{TCA} + \Delta E_{clutter}$$

Donde:
- 139.3 = constante normalizadora ITU (20·log₁₀(10⁷))
- $E$ = intensidad de campo desde tablas [dBμV/m]
- $\Delta E_{TCA}$ = corrección terrain clearance angle
- $\Delta E_{clutter}$ = corrección ambiente

```python
def _convert_field_to_path_loss(self, E_field, freq_mhz, tca_corr, clutter_corr):
    path_loss = (139.3 
                 + 20 * np.log10(freq_mhz) 
                 - E_field 
                 + tca_corr 
                 + clutter_corr)
    return path_loss
```

## 4. Corrección de Percentil (§8.1)

**Método:** `_apply_percentile_correction(percentile, freq_mhz, distances_km)`

P.1546-6 §8.1 define correcciones por percentil de tiempo en **suma algebraica**
(no cuadrática) aplicada como ajuste de campo `ΔE_percentile [dB]`.

Los únicos percentiles con tablas en el estándar ITU son: **{1, 10, 50, 90, 99}**.
Los percentiles intermedios se redondean al valor de tabla más cercano disponible:

```python
percentile_map = {1: 1, 5: 10, 10: 10, 25: 10, 50: 50, 75: 90, 90: 90, 95: 90, 99: 99}
```

La corrección se suma algebraicamente al campo antes de la conversión E→PL:

$$PL = 139.3 + 20\log_{10}(f) - (E + \Delta E_{TCA} + \Delta E_{clutter} + \Delta E_{percentile})$$

## 5. Método Informativo: `_calculate_radio_horizon()`

**Fórmula estándar** (Radio Engineering, k = 4/3):

$$d_{ho} = 4.12\,(\sqrt{h_{TX}} + \sqrt{h_{RX}}) \quad [\text{km}]$$

```python
def _calculate_radio_horizon(self, tx_height: float, rx_height: float) -> float:
    return 4.12 * (math.sqrt(tx_height) + math.sqrt(rx_height))
```

> **Importante:** Este método **NO se invoca** dentro de `calculate_path_loss()`.
> P.1546-6 no usa el radio horizonte explícitamente en su procedimiento de cálculo
> de path loss. El método existe como utilidad informativa (referencia de cobertura
> teórica máxima LOS), pero el pipeline del modelo opera íntegramente con la
> interpolación de tablas y la corrección TCA continua.
