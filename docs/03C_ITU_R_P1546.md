## 1. Descripción del Modelo

El modelo **ITU-R P.1546** es un modelo **punto-a-área empírico** diseñado para predecir la intensidad de campo o pérdida de propagación en comunicaciones terrestres. La recomendación P.1546 se basa en una base de datos extensa de mediciones en bandas VHF, UHF y SHF en condiciones reales (tierra, mar, mixto).

### 1.1 Características Distintivas

A diferencia de Okumura-Hata o COST-231 que son específicos para sistemas celulares, P.1546 fue diseñado para:
- Radiodifusión (broadcasting) VHF/UHF
- Sistemas punto-a-área de largo alcance
- Predicciones estadísticas de cobertura (50% de tiempo, 50% de ubicaciones)

La implementación en este sistema adapta la recomendación para planificación de cobertura celular.

### 1.2 Tipo de Modelo

| Aspecto | Descripción |
|---------|-------------|
| Tipo | Empírico punto-a-área (estadístico) |
| Condición LOS/NLOS | Sí — radio horizon distance |
| Uso de terreno | Sí — 3 tipos de terreno, altura efectiva TX |
| Vectorización | CPU (NumPy) / GPU (CuPy) via `self.xp` |
| Referencia | ITU-R P.1546-6 (agosto 2019) |

### 1.3 Rangos de Validez

| Parámetro | Mínimo | Máximo | Unidades |
|-----------|--------|--------|----------|
| Frecuencia | 30 | 4000 | MHz |
| Distancia | 1 | 1000 | km |
| Altura TX | 10 | 3000 | m AGL |
| Altura RX | 1 | 20 | m AGL |

## 2. Entrada y Salida del Método Principal

**Ubicación:** `itu_r_p1546.py`, método `calculate_path_loss()`

### 2.1 Parámetros de Entrada

```python
def calculate_path_loss(
    self,
    distances,             # np.ndarray, metros, shape (N,) o (H, W)

```

| Parámetro | Tipo | Unidades | Descripción | Default |
|-----------|------|----------|-------------|---------|
| `distances` | ndarray | m | Distancias Haversine TX→receptor | — |

| `tx_height` | float | m AGL | Altura antena sobre suelo local TX | — |
| `terrain_heights` | ndarray | msnm | Elevación terreno en cada punto receptor | — |

| terrain_type | Descripción | Corrección (dB) |
|-------------|-------------|-----------------|
| `'smooth'` | Agua, llanura plana | −2 dB (ganancia) |

## 3. Estructura de la Fórmula Principal

La pérdida de propagación total se calcula como suma de cuatro componentes:

$$\boxed{PL = L_0 + \Delta_h + \Delta_f + \Delta_{\text{env}}}$$

donde cada componente se calcula en un método separado:

| Componente | Método | Descripción |
|-----------|--------|-------------|
| $L_0$ | `_calculate_base_path_loss()` | Pérdida en espacio libre (baseline) |
| $\Delta_h$ | `_calculate_height_correction()` | Corrección por alturas TX y RX |
| $\Delta_f$ | `_calculate_frequency_correction()` | Corrección para f > 300 MHz |
| $\Delta_{\text{env}}$ | `_calculate_environment_correction()` | Corrección por ambiente y terreno |

**Código real (ensamblaje):**
```python
# itu_r_p1546.py, método calculate_path_loss()
pl_base     = self._calculate_base_path_loss(frequency, distances_km)
is_los      = distances_km <= radio_horizon_km
height_corr = self._calculate_height_correction(frequency, tx_height,
                mobile_height, distances_km, is_los)
freq_corr   = self._calculate_frequency_correction(frequency, distances_km)
env_corr    = self._calculate_environment_correction(frequency, distances_km,
                environment, terrain_type, is_los)

path_loss = pl_base + height_corr + freq_corr + env_corr
```

## 4. Integración con el Terreno

### 4.1 Altura Efectiva TX

La altura efectiva del TX se usa en el cálculo del radio horizon:

$$h_{\text{tx,eff}} = h_{\text{ant}} + h_{\text{elev,TX}} - \overline{h_{\text{terreno}}}$$

**Código real:**
```python
# itu_r_p1546.py, líneas 161-162
tx_height_eff = tx_height + tx_elevation - self.xp.mean(terrain_heights_flat)
```

A diferencia de Okumura-Hata y COST-231, el modelo **no aplica clipping** a `h_tx_eff` — el rango válido es mucho más amplio (10–3000 m).

### 4.2 Impacto del Tipo de Terreno

El tipo de terreno se incorpora en $\Delta_{\text{env}}$ con una corrección fija:

```python
# itu_r_p1546.py, _calculate_environment_correction()
if terrain_type.lower() == 'smooth':
    terrain_correction = -2.0      # propagación favorable (agua, llanura)
elif terrain_type.lower() == 'mixed':
    terrain_correction = 0.0       # referencia
elif terrain_type.lower() == 'irregular':
    terrain_correction = 3.0       # propagación desfavorable (montañas)
```

## 5. Componente 1: Pérdida en Espacio Libre (L₀)

$$L_0 = 20\,\log_{10}(f_{\text{MHz}}) + 20\,\log_{10}(d_{\text{km}}) + 32.45 \quad \text{[dB]}$$

Esta ecuación es idéntica a la del modelo Free Space (ver [03E_FREE_SPACE_v2.md](03E_FREE_SPACE_v2.md)). Sirve como punto de partida (baseline) al que se suman las correcciones del entorno real.

**Código real:**
```python
# itu_r_p1546.py, método _calculate_base_path_loss()
distances_km_safe = self.xp.maximum(distances_km, 0.001)  # protección log(0)
l0 = (20 * self.xp.log10(frequency)
      + 20 * self.xp.log10(distances_km_safe)
      + 32.45)
```

## 6. Determinación LOS/NLOS — Radio Horizon

### 6.1 Fórmula del Radio Horizon

La frontera LOS/NLOS se determina por la **distancia de radio horizon**, que depende de las alturas de TX y RX, considerando la curvatura terrestre con factor de tierra efectiva k=4/3:

$$d_{\text{ho}} = \frac{4.12\,\sqrt{h_{\text{tx}} \cdot h_{\text{rx}}}}{100} \quad \text{[km]}$$

donde las alturas están en **metros AGL**.

**Código real:**
```python
# itu_r_p1546.py, método _calculate_radio_horizon()
d_ho = 4.12 * self.xp.sqrt(tx_height * rx_height) / 100.0
return float(d_ho)
```

**Justificación matemática:** La distancia de radio horizon para una antena a altura h sobre una Tierra esférica con radio efectivo $R_e = k\,R_0$ (k=4/3 para atmósfera estándar) es:

$$d = \sqrt{2\,R_e\,h} \approx \sqrt{2 \times 8500 \times h} = 4.12\,\sqrt{h} \quad \text{[km, h en m]}$$

Para dos antenas:
$$d_{\text{ho}} = \sqrt{2\,R_e\,h_{\text{tx}}} + \sqrt{2\,R_e\,h_{\text{rx}}} \approx 4.12\,(\sqrt{h_{\text{tx}}} + \sqrt{h_{\text{rx}}})$$

La implementación usa el producto en lugar de la suma: `4.12·sqrt(h_tx·h_rx)/100` — es una aproximación compacta que da valores similares en el rango típico.

### 6.2 Clasificación LOS/NLOS

```python
# itu_r_p1546.py, línea 155
radio_horizon_km = self._calculate_radio_horizon(tx_height, mobile_height)
is_los = distances_km <= radio_horizon_km
# → array booleano: True = LOS, False = NLOS
```

**Ejemplo:** Para tx_height=50 m, rx_height=1.5 m:
```
d_ho = 4.12 × sqrt(50 × 1.5) / 100
     = 4.12 × sqrt(75) / 100
     = 4.12 × 8.66 / 100
     = 0.357 km ≈ 357 m
→ puntos < 357 m: LOS
→ puntos > 357 m: NLOS
```

## 7. Componente 2: Corrección por Altura (Δh)

Este componente captura el efecto de las alturas de TX y RX sobre la propagación. Valores **negativos** representan ganancia (reducen el path loss); valores **positivos** representan atenuación.

### 7.1 Corrección por Altura del Receptor (RX)

Normalizada a 10 m de referencia:

$$\Delta h_{\text{rx}} = -20\,\log_{10}\!\left(\frac{h_{\text{rx}}}{10}\right) \quad \text{[dB]}$$

**Interpretación:**
- h_rx = 10 m → Δh_rx = 0 dB (referencia)
- h_rx = 1.5 m → Δh_rx = −20·log₁₀(0.15) = +16.5 dB (receptor bajo → más pérdida)
- h_rx = 20 m → Δh_rx = −6.02 dB (receptor alto → ganancia)

> **Nota de signo:** El código usa `−20·log10(h_rx/10)`. Si h_rx < 10 m, el argumento es < 1, log es negativo, y el término completo es positivo → aumenta la pérdida. Esto es físicamente correcto: un receptor bajo tiene más difracción y obstáculos.

**Código real:**
```python
# itu_r_p1546.py, _calculate_height_correction()
rx_correction = -20 * self.xp.log10(rx_height / 10.0)
```

### 7.2 Corrección por Altura del Transmisor (TX)

Normalizada a 200 m de referencia, diferenciada por LOS/NLOS:

$$\Delta h_{\text{tx}} = \begin{cases}
-20\,\log_{10}\!\left(\frac{h_{\text{tx}}}{200}\right) - \delta_d & \text{LOS} \\[6pt]
-10\,\log_{10}\!\left(\frac{h_{\text{tx}}}{200}\right) - 0.5\,\delta_d & \text{NLOS}
\end{cases}$$

donde $\delta_d = \log_{10}(\max(d_{\text{km}}, 1)) = $ factor de distancia normalizado.

**Código real:**
```python
# itu_r_p1546.py, _calculate_height_correction()
tx_factor = self.xp.log10(tx_height / 200.0)
distance_factor = 10 * self.xp.log10(self.xp.maximum(distances_km, 1.0)) / 10.0

tx_correction = self.xp.where(
    is_los,
    -20 * tx_factor - distance_factor,       # LOS: efecto completo
    -10 * tx_factor - 0.5 * distance_factor  # NLOS: efecto reducido
)
```

**Justificación:** En LOS, la altura del TX tiene efecto completo de 20 dB/dec (similar a Free Space). En NLOS, los obstáculos reducen el beneficio a 10 dB/dec. La altura de referencia de 200 m proviene de las curvas de campo eléctrico de P.1546 (400 MHz, 50 km).

### 7.3 Corrección Total

$$\Delta_h = \Delta h_{\text{rx}} + \Delta h_{\text{tx}} \quad \text{[dB]}$$

## 8. Componente 3: Corrección por Frecuencia (Δf)

Para frecuencias superiores a 300 MHz, P.1546 aplica una corrección adicional:

$$\Delta_f = \begin{cases}
10\,\log_{10}(f/1000) & \text{si } f > 300 \text{ MHz} \\
0 \text{ dB} & \text{si } f \leq 300 \text{ MHz}
\end{cases}$$

**Código real:**
```python
# itu_r_p1546.py, _calculate_frequency_correction()
if frequency > 300:
    delta_f = 10 * self.xp.log10(max(frequency, 100) / 1000.0)
else:
    delta_f = 0.0
```

**Justificación:** Para f > 300 MHz, las curvas de referencia de P.1546 (originalmente tabuladas en 100, 600, 2000 MHz) requieren interpolación entre bandas. Esta corrección es una aproximación a esa interpolación.

**Ejemplos:**
| f (MHz) | Δf (dB) |
|---------|---------|
| 150 | 0.0 |
| 300 | 0.0 |
| 900 | 10·log10(0.9) = −0.46 |
| 1800 | 10·log10(1.8) = +2.55 |
| 2600 | 10·log10(2.6) = +4.15 |

## 9. Componente 4: Corrección por Ambiente y Terreno (Δenv)

### 9.1 Corrección por Ambiente

$$\Delta_{\text{amb}} = \begin{cases}
0 \text{ dB} & \text{Urban (referencia)} \\
-2 - 3\,\log_{10}(f/1000) & \text{Suburban} \\
-4 - 5\,\log_{10}(f/1000) & \text{Rural}
\end{cases}$$

**Código real:**
```python
# itu_r_p1546.py, _calculate_environment_correction()
freq_factor = self.xp.log10(max(frequency, 50) / 1000.0)

if environment.lower() == 'urban':
    env_correction = 0.0
elif environment.lower() == 'suburban':
    env_correction = -2.0 - 3.0 * freq_factor
elif environment.lower() == 'rural':
    env_correction = -4.0 - 5.0 * freq_factor
```

**Ejemplos a 900 MHz** (freq_factor = log10(0.9) = −0.046):
| Ambiente | Δ_amb |
|---------|-------|
| Urban | 0.0 dB |
| Suburban | −2.0 − 3·(−0.046) = −1.86 dB |
| Rural | −4.0 − 5·(−0.046) = −3.77 dB |

### 9.2 Corrección por Terreno

| terrain_type | Corrección |
|-------------|-----------|
| smooth | −2.0 dB |
| mixed | 0.0 dB |
| irregular | +3.0 dB |

### 9.3 Factor LOS/NLOS

La corrección total de ambiente+terreno se escala según la condición LOS/NLOS:

$$\Delta_{\text{env}} = (\Delta_{\text{amb}} + \Delta_{\text{terreno}}) \times \alpha_{\text{LOS/NLOS}}$$

$$\alpha = \begin{cases} 0.7 & \text{LOS (efecto reducido)} \\ 1.0 & \text{NLOS (efecto completo)} \end{cases}$$

**Código real:**
```python
# itu_r_p1546.py, _calculate_environment_correction()
correction_factor = self.xp.where(is_los, 0.7, 1.0)
delta_env = (env_correction + terrain_correction) * correction_factor
```

**Justificación:** En LOS, la señal viaja directamente — el ambiente circundante tiene menos impacto. En NLOS, la difracción obliga a la señal a interactuar más con el entorno local.

## 10. Flujo de Cálculo Completo

```
Entrada: distances (m), frequency (MHz), tx_height (m),
         terrain_heights (msnm), tx_elevation (msnm),
         environment, terrain_type, mobile_height

     │
     ▼
┌────────────────────────────────────────────────────────┐
│ 1. Flatten + validación                               │
│    distances_flat = flatten(distances)                │
│    terrain_flat = flatten(terrain_heights)            │
└────────────────────────────────────────────────────────┘
     │
     ▼
┌────────────────────────────────────────────────────────┐
│ 2. Conversión                                         │
│    distances_km = distances / 1000                    │
└────────────────────────────────────────────────────────┘
     │
     ▼
┌────────────────────────────────────────────────────────┐
│ 3. L0 (Free Space baseline)                          │
│    L0 = 32.45 + 20·log(f) + 20·log(d_km)            │
└────────────────────────────────────────────────────────┘
     │
     ▼
┌────────────────────────────────────────────────────────┐
│ 4. Radio Horizon → LOS/NLOS                          │
│    d_ho = 4.12·√(h_tx·h_rx) / 100  [km]             │
│    is_los = (distances_km ≤ d_ho)                    │
└────────────────────────────────────────────────────────┘
     │
     ▼
┌────────────────────────────────────────────────────────┐
│ 5. Δh — Corrección por alturas                       │
│    Δh_rx = −20·log10(h_rx/10)                        │
│    tx_factor = log10(h_tx/200)                        │
│    dist_factor = log10(max(d_km,1))                  │
│    LOS:  Δh_tx = −20·tx_factor − dist_factor         │
│    NLOS: Δh_tx = −10·tx_factor − 0.5·dist_factor     │
│    Δh = Δh_rx + Δh_tx                                │
└────────────────────────────────────────────────────────┘
     │
     ▼
┌────────────────────────────────────────────────────────┐
│ 6. Δf — Corrección por frecuencia                    │
│    f > 300: Δf = 10·log10(f/1000)                    │
│    f ≤ 300: Δf = 0                                   │
└────────────────────────────────────────────────────────┘
     │
     ▼
┌────────────────────────────────────────────────────────┐
│ 7. Δenv — Corrección por ambiente+terreno             │
│    Δamb = f(environment, freq_factor)                 │
│    Δterr = f(terrain_type) ∈ {−2, 0, +3} dB         │
│    α = 0.7 (LOS) o 1.0 (NLOS)                        │
│    Δenv = (Δamb + Δterr) × α                         │
└────────────────────────────────────────────────────────┘
     │
     ▼
┌────────────────────────────────────────────────────────┐
│ 8. PL = L0 + Δh + Δf + Δenv                         │
│    reshape a original_shape                           │
└────────────────────────────────────────────────────────┘
     │
     ▼
Salida: path_loss (dB), shape = shape(distances)
```

## 11. Ejemplo Numérico Completo

**Escenario:** Sistema de radiodifusión VHF, entorno rural, terreno irregular

```
Parámetros:
  frequency     = 200 MHz
  tx_height     = 100 m AGL
  mobile_height = 5 m AGL
  distance      = 20 km (en zona NLOS)
  environment   = 'Rural'
  terrain_type  = 'irregular'

Paso 3 — L0:
  L0 = 20·log10(200) + 20·log10(20) + 32.45
     = 20·2.301 + 20·1.301 + 32.45
     = 46.02 + 26.02 + 32.45
     = 104.5 dB

Paso 4 — Radio Horizon:
  d_ho = 4.12·√(100×5) / 100
       = 4.12·√500 / 100
       = 4.12·22.36 / 100 = 0.921 km
  20 km > 0.921 km → NLOS

Paso 5 — Δh (NLOS):
  Δh_rx = −20·log10(5/10) = −20·(−0.301) = +6.02 dB
  tx_factor = log10(100/200) = log10(0.5) = −0.301
  dist_factor = log10(max(20,1)) = log10(20) = 1.301
  Δh_tx (NLOS) = −10·(−0.301) − 0.5·1.301 = +3.01 − 0.65 = +2.36 dB
  Δh = 6.02 + 2.36 = 8.38 dB

Paso 6 — Δf (f=200 ≤ 300):
  Δf = 0 dB

Paso 7 — Δenv (NLOS, Rural, Irregular, α=1.0):
  freq_factor = log10(200/1000) = log10(0.2) = −0.699
  Δamb (Rural) = −4 − 5·(−0.699) = −4 + 3.495 = −0.505 dB
  Δterr (Irregular) = +3.0 dB
  Δenv = (−0.505 + 3.0) × 1.0 = 2.495 dB

Resultado:
  PL = 104.5 + 8.38 + 0 + 2.495
     = 115.4 dB
```

## 12. Comparación LOS vs NLOS para el Mismo Punto

Para la antena anterior (f=200 MHz, h_tx=100 m) evaluando a 0.5 km (LOS) vs 20 km (NLOS):

| Componente | LOS (0.5 km) | NLOS (20 km) |
|-----------|-------------|-------------|
| L0 | 32.45 + 46.02 + (−6.02) = 72.5 dB | 104.5 dB |
| Δh_rx | +6.02 dB | +6.02 dB |
| Δh_tx | LOS: −20·(−0.301)−(−0.65)=+6.67 dB | NLOS: +2.36 dB |
| Δf | 0 | 0 |
| Δenv | α=0.7: 2.495×0.7=1.75 dB | α=1.0: 2.50 dB |
| **PL Total** | **86.9 dB** | **115.4 dB** |

Diferencia: ~28.5 dB entre LOS cercano y NLOS lejano.
